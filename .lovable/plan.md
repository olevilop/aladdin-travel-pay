# Замена автономера на дату создания

Сейчас при создании карточки генерируется номер вида `2026-0001`. Заменяем на дату создания в формате `ДД.ММ.ГГГГ`, которая подставляется автоматически в момент создания.

## Изменения

**1. `src/lib/api.ts` → `createApplication` (мок-ветка)**
- Убрать вызов `newAppNumber()`.
- В поле `number` записывать `formatDate(nowISO())` (импорт из `@/lib/format`).

**2. `src/lib/mock/store.ts`**
- В seed-данных заменить `number: "2026-0001"`, `"2026-0002"`, `"2026-0003"` на соответствующие даты: `"01.05.2026"`, `"10.05.2026"`, `"18.05.2026"` (берутся из `created_at` тех же записей).
- Функцию `newAppNumber()` и связанную с ней `nextApplicationNumber` в `src/lib/format.ts` можно удалить (больше не используются) — оставлю удаление, чтобы не было мёртвого кода.

**3. `src/components/new-application-dialog.tsx`**
- Текст `DialogDescription` поменять с «Номер сгенерируется автоматически в формате 2026-0001.» на «Дата создания подставится автоматически.»
- `toast.success` после создания: вместо `Создана заявка ${app.number}` → `Создана заявка от ${app.number}` (т.к. number теперь = дата).

**4. `src/routes/applications.index.tsx` (карточка в списке)**
- Сверху слева: оставить `{app.number}` (теперь это дата), убрать префикс `№ `.
- Снизу справа: убрать `{formatDate(app.created_at)}`, оставить только `{app.files.length} файлов` (чтобы не дублировать дату).

**5. `src/routes/applications.$id.tsx` (страница заявки)**
- Проверить заголовок/хлебные крошки: если там есть `№ {number}` — заменить на просто `{number}` (дата), либо `Создано: {number}`. Точный текст уточню при правке файла.

## Поле `number` в типах
Тип `Application.number: string` менять не нужно — формат строки просто меняется с `2026-0001` на `ДД.ММ.ГГГГ`. Это минимизирует диффы и сохраняет совместимость с реальным backend (если поле когда-то заполнит сервер — оно тоже строка).

## Что не трогаем
- Поле `created_at` (ISO timestamp) остаётся — оно нужно для сортировки списка.
- Сортировка по `created_at desc` в `listApplications` сохраняется.
- Поиск по `number` продолжит работать — можно будет искать по дате (например, `23.05`).
